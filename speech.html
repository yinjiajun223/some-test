解决本地开发麦克风权限问题

在本地开发环境中，浏览器出于安全考虑通常会阻止麦克风权限访问。本页面提供多种解决方案，并包含一个语音转文字功能演示。

问题描述

当你在本地文件系统（file://）或HTTP协议下开发时，现代浏览器会阻止网页访问麦克风等敏感设备。

这是因为浏览器实施的安全策略要求使用HTTPS协议才能访问这些API（除了localhost之外）。

错误信息通常为：NotAllowedError（Permission denied）或只在安全上下文中允许使用这些功能。

完整解决方案

以下是解决本地开发麦克风权限问题的可靠方法：

  1. 使用localhost开发 - 通过本地服务器（如Live Server）运行你的应用
  2. 配置HTTPS - 为本地开发环境设置自签名证书
  3. 修改浏览器设置 - 临时允许不安全的来源（仅用于开发）
  4. 使用开发工具 - 如ngrok或localhost.run创建安全隧道
# 使用Python启动本地服务器
python -m http.server

# 安装和使用live-server
npm install -g live-server
live-server --port=8080
